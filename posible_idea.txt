El objetivo principal del proyecto es desarrollar un modelo predictivo capaz de estimar la vida útil restante de motores turbofan a partir de mediciones históricas de sensores y condiciones operativas. Para ello se utilizará el dataset NASA C-MAPSS, que contiene series temporales simuladas del proceso de degradación de motores hasta alcanzar la falla. La tarea busca abordar un problema de mantenimiento predictivo: dado el estado actual de un motor y su comportamiento reciente, estimar cuántos ciclos operativos le quedan antes de fallar.

El problema se plantea principalmente como una tarea supervisada de regresión, ya que la variable objetivo será el Remaining Useful Life (RUL), expresado como una cantidad numérica de ciclos restantes. Aunque el RUL toma valores enteros, estos valores poseen orden y distancia entre sí, por lo que resulta más adecuado tratarlos como una magnitud continua que como clases independientes.

Antes de definir la configuración final de los modelos, se realizará un análisis exploratorio de los datos para estudiar la evolución de los sensores, detectar variables constantes o poco informativas, analizar posibles correlaciones entre mediciones y evaluar la influencia de las condiciones operativas. Este análisis permitirá decidir qué sensores conservar, qué transformaciones aplicar y qué variables temporales derivadas construir.

Como arquitectura baseline se propone utilizar una Multilayer Perceptron (MLP) simple. Esta elección se justifica porque las variables de entrada son numéricas y porque una MLP permite modelar relaciones no lineales entre sensores, condiciones operativas y degradación del motor sin introducir una complejidad excesiva. El baseline utilizará principalmente las mediciones disponibles en cada ciclo, por lo que servirá como punto de comparación inicial para evaluar cuánto puede aprender el modelo a partir del estado puntual del motor.

Luego se evaluará una MLP de mayor capacidad, incorporando regularización y variables derivadas mediante feature engineering temporal. Esta variante no se plantea como una garantía de mejora, sino como una forma de comprobar si agregar información sobre la evolución reciente de los sensores aporta valor predictivo. Para ello se construirán variables como promedios móviles, diferencias entre ciclos, tendencias o variabilidad en ventanas temporales. La justificación es que el deterioro de un motor no depende únicamente de una medición aislada, sino de cómo cambian sus señales a lo largo del tiempo.

Como arquitectura adicional experimental, se evaluará un autoencoder determinístico seguido de una MLP regresora. El autoencoder buscará aprender una representación comprimida del estado del motor a partir de los sensores, especialmente si se observa redundancia o correlación entre variables. Luego, esa representación latente se utilizará como entrada para predecir el RUL. Esta variante se incluirá como comparación acotada, no como reemplazo del modelo principal.
